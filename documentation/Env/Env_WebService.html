		doRequest (line 91)
	
 
	
	

La méthode qui exécute la requête.


 Si une erreur apparaît, on ferme l'appel CURL et met les détails de l'erreur dans la variable $this->errorText. 
On distingue 2 situations où en retour on reçoit le code 404 :   

1) API renvoie le code 404 pour une requête correcte (vers une ressource existante)
qui ne contient pas les résultats.  Dans ce cas le type de la réponse est application/xml.   

2) Le serveur renvoie le code 404. Il fait cela pour une ressource inexistante, comme chaque page 404.
Dans cette situation, le type de la réponse est text/html.   

Si la réponse du serveur est 404, on annule l'opération en mettant $result à false, $respError à true,
et en rajoutant un message d'erreur à $respErrorsList avec "http_file_not_found" comme clé.

Si la réponse de l'API est 404, on n'annule pas l'opération. On appelle la méthode setResponseError().
qui se charge de remplir le tableau contenant les messages d'erreur.

		parseResponse (line 156)
	
 
	
	

La méthode qui parse la réponse du serveur.



Au début on vérifie si la réponse ne contient pas la balise <error /> . 
Si elle ne la contient pas, on ne fait rien. 

Sinon, on change $respError en true, parse la réponse et 
définit les messages d'erreur dans le tableau $respErrorsList.
